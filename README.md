# Swiggy_Sales_Analysis

Welcome to the Swiggy Sales Data Warehouse & Analytics Project repository.
This project demonstrates an end-to-end SQL analytics workflow, covering data validation, dimensional modeling, and business insight generation using Microsoft SQL Server. It is designed as a portfolio project showcasing industry-aligned best practices in data engineering and analytics.

---

### Objective
Data Cleaning & Data Transformation of the Swiggy sales dataset, Normalization of the dataset and answering business questions

#### Specifications
- **Data Source**: Raw Swiggy sales dataset imported into SQL Server
- **Data Quality**: Validation and cleaning to handle nulls, blanks, and duplicates
- **Modeling**: Star schema design with fact and dimension tables
Integration: Centralized fact table linked to supporting dimensions for analytical queries

---

### BI: Analytics & Reporting (Data Analytics)

#### Objective
Develop SQL-based analysis to generate actionable insights into:
- **Order and Revenue Trends**
- **Geographic Performance (State and City)**
- **Restaurant and Cuisine Performance**
  
These insights support data-driven decision-making and performance evaluation..
